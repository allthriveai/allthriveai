# Profile Redesign: Asks + Offers Side-by-Side

> **Last Updated:** January 2025
>
> This document defines the new profile layout for the Ask & Offer AI Collective.

---

## Design Philosophy

Profiles are **creative hubs** that prominently feature **Asks and Offers side-by-side**, while also showcasing the person's work and interests.

**Key insight:** An app isn't an offer â€” it's a thing. The *offer* is what you do with it (mentoring, consulting, courses). The *ask* is what you need for it (testers, feedback, collaborators). By separating "Things I'm Working On" from Asks/Offers, we avoid forcing creative content into transactional boxes.

**Structure:**
1. **Header** â€” Name, tagline, stats, social links
2. **Asks + Offers** â€” Side-by-side columns (the core marketplace)
3. **Things I'm Working On or Thinking About** â€” Projects, apps, WIPs, ideas, blog posts (creative showcase)
4. **Things I Love** â€” Curated links and recommendations
5. **Recent Activity** â€” Shows engagement
6. **Achievements** â€” Gamification/badges

**Key principles:**
- Asks and Offers are **equal and prominent** â€” side-by-side, not sequential
- Creative content (apps, WIPs, ideas, blog posts) lives in its own section
- Each project card shows its linked asks and offers
- Empty sections are hidden automatically

### Category Flexibility

Categories are **not hardcoded** â€” users can add custom categories. The matching system uses:
1. **Category match** â€” Main category pairing
2. **Tag overlap** â€” Shared tags (AI, Marketing, etc.)
3. **Tool overlap** â€” Both use Claude, etc.
4. **Recency** â€” Newer = more relevant
5. **Reputation** â€” Helped count, ratings

## Data Model

### Project Model (Things I'm Working On)

```python
class Project(models.Model):
    PROJECT_TYPES = [
        ('app', 'App'),
        ('agent', 'Agent'),
        ('mcp', 'MCP Server'),
        ('post', 'Blog Post'),
        ('game', 'Game'),
        ('idea', 'Idea'),
        ('wip', 'Work in Progress'),
    ]

    STATUS_CHOICES = [
        ('live', 'Live'),
        ('coming_soon', 'Coming Soon'),
        ('wip', 'Work in Progress'),
        ('idea', 'Idea'),
    ]

    user = models.ForeignKey(User, on_delete=models.CASCADE, related_name='projects')
    title = models.CharField(max_length=200)
    description = models.TextField(blank=True)
    project_type = models.CharField(max_length=20, choices=PROJECT_TYPES)
    status = models.CharField(max_length=20, choices=STATUS_CHOICES, default='wip')
    featured_image_url = models.URLField(blank=True)
    url = models.URLField(blank=True)  # Link to try it / read it
    tags = models.ManyToManyField('taxonomy.Taxonomy', blank=True)
    tools = models.ManyToManyField('tools.Tool', blank=True)
    created_at = models.DateTimeField(auto_now_add=True)
```

## Key Design Decisions

1. **Asks and Offers are equal** â€” Neither is subordinate. Side-by-side layout emphasizes this.

2. **Stats show asks and offers counts** â€” Reinforces the marketplace identity.

3. **Creative content is separate** â€” Apps, WIPs, blog posts, ideas don't need to be "offers." They live in "Things I'm Working On or Thinking About."

4. **Projects link to asks/offers** â€” Each project card shows its attached asks ("Need testers") and offers ("Consulting available").

5. **Right-side tray for all actions** â€” Consistent interaction pattern. CTAs open a tray with message composer and optional payment.

6. **Categories are flexible** â€” Users can add custom categories. Matching uses categories + tags + tools.

7. **SMS notifications** â€” All connections trigger SMS to recipient for immediate attention.
